# Projektcode

Hier entsteht der eigentliche Code des Projekts.

Alle Dateien, Ordner und Unterordner, die zum Projekt gehören, kommen ausschließlich
in diesen Bereich. Nichts davon gehört in den Root oder in andere Hauptordner.

## Konventionen

- Secrets und `.env`-Dateien bleiben lokal und werden nicht versioniert.
- `node_modules/`, `dist/`, Logs und Build-Ausgaben kommen in `.gitignore`.
- Architekturentscheidungen werden in `DOKUMENTATION/Informationen/Entscheidungen.md` festgehalten.
- Offene Punkte gehören nach `DOKUMENTATION/ToDo/Offene_Punkte.md`.
